# monitoring

Собственный агент наблюдаемости для сети relay-узлов.

Зачем свой, а не готовый стек: нужен был полный контроль над тем, что измеряется
(доступность, RTT, аптайм, состояние воркеров на узле), при минимальном оверхеде —
без разворачивания и обслуживания тяжёлого Prometheus/Grafana на раннем этапе.

**Что делает**
- TCP-пробы каждого узла с заданным интервалом;
- считает аптайм и хранит короткую историю состояний;
- выводит сводку (в проде — отдаёт метрики в бэкенд).

**Конфигурация** — через переменные окружения (`.env`):
- `MONITOR_TARGETS` — узлы `host:port` через запятую;
- `MONITOR_INTERVAL_SEC` — период опроса.
